| :--- | :--- | :--- |
| **Professor Zap** | Planner | High-level strategy and task decomposition |
| **Patch** | Implementer | Code generation and feature execution |
| **Reroute** | Debugger | Error analysis and automated fixing |
| **Firewall Frankie** | Security | Vulnerability scanning and policy enforcement |
| **Semver Sam** | Release | Versioning, changelogs, and deployment |

## Getting Started
### Initialize a Repository
To bootstrap a new GLITCHLAB environment in your current directory:
```bash
glitchlab init
```

## Architecture
GLITCHLAB is built on a modular foundation:
- **governance/**: Orchestration logic, agent definitions, and decision-making frameworks.
- **workspace/**: File system abstraction, environment isolation, and tool integration.
